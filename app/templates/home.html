Pear Programmer
  • Signup/Login

The first, truly conversational and automated coding assistant.

Give Pear a list of tasks and watch it work its magic while you indulge in Reddit scrolling or obsessively checking crypto prices. It's like having a super productive sidekick who builds entire features for you, while you perfect the art of procrastination.

Watch the Demo


Getting Started

  1. Create an account and get an API key
  2. Make sure your system has ruby installed
  3. Run gem install pear-programmer
  4. Run pear-on init to initialize the CLI (from the root of your project).

Note, Pear currently supports Rails and Python applications.

Usage

pear-on help # to show help menu
pear-on coding new # create a new coding task
pear-on coding start # run your coding task 
-a --auto # will only ask for your response when necessary
pear-on coding list # view your tasks

Prompting

  • Context when initializing Pear you want to provide sufficient information about your application so the LLM has a good starting point.
    • The type of application, ie Ruby on Rails
    • The type of database you are using, ie Postgres
    • If you are using a package manager, ie Yarn, and what packages you are using, ie Bootstrap for styles
    • The structure of your application, ie all business logic goes in app/services
    • And any other information you think will make the LLM more efficient
  • Requirements a high-level overview of what you are trying to accomplish. A sentence or two should suffice.
    • For example, if you are creating a dashboard you can say "Building a dashboard that allows users to manage credit cards"
  • Tasks these are steps that the LLM should perform in order to accomplish the requirements. You should be as specific as possible to make the LLM most efficient.
    • For example, review app/views.py
    • Add authentication for the DashboardView
    • Review app/templates/dashboard.html
    • Add a section in app/templates/dashboard.html to display the signed in user's name
